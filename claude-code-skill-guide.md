# Claude Code 스킬(Skill) 만들기 — 공부 가이드

> 개인 학습용 정리 노트. Claude Code의 스킬이 무엇인지 이해하고, 직접 만들어보기 위한 시작 가이드.

---

## 1. 스킬(Skill)이란?

Claude Code의 **스킬**은 "특정 작업을 할 때 따라야 할 지침과 도구를 담은 폴더"다. 핵심은 딱 하나의 파일, `SKILL.md`이다.

```
<skill-name>/
├── SKILL.md          ← 필수. 스킬의 정의와 지침
├── (선택) scripts/    ← 스킬이 실행할 스크립트
├── (선택) templates/  ← 참고용 템플릿
└── (선택) reference/  ← 추가 참고 문서
```

### SKILL.md의 구조

```markdown
---
name: my-skill
description: 이 스킬을 "언제" 써야 하는지 설명. 자동 호출의 핵심 기준.
---

여기에 Claude가 따라야 할 구체적인 지침을 마크다운으로 작성한다.
```

### 동작 방식 (두 가지 호출 경로)

| 호출 방식 | 설명 |
|---|---|
| **수동 호출** | 사용자가 `/my-skill` 처럼 슬래시 명령으로 직접 실행 |
| **자동 호출** | Claude가 `description`을 읽고 "지금 이 작업에 맞다"고 판단하면 스스로 사용 |

> 그래서 `description`이 **가장 중요**하다. "무엇을, 언제, 어떤 상황에서 쓰는지"를 구체적으로 적어야 Claude가 제때 꺼내 쓴다.

### 저장 위치

| 위치 | 범위 |
|---|---|
| `~/.claude/skills/<name>/SKILL.md` | 개인용 — 모든 프로젝트에서 사용 |
| `.claude/skills/<name>/SKILL.md` | 프로젝트용 — 해당 프로젝트 + 팀 공유(git 커밋) |

### 스킬의 강점

- **재사용**: 반복 작업의 절차를 한 번 정의해두고 계속 호출
- **점진적 로딩(progressive disclosure)**: SKILL.md는 짧게 두고, 상세 내용은 별도 파일로 두어 필요할 때만 읽게 할 수 있음 → 컨텍스트 절약
- **도구 제한**: 필요하면 `allowed-tools`로 그 스킬이 쓸 수 있는 도구를 제한 가능

## 2. 시작 To-Do List

### 기획 단계
- [ ] **목적 정의** — 이 스킬이 해결할 작업 한 문장으로 (예: "PR 리뷰 코멘트를 한국어로 요약")
- [ ] **트리거 정의** — 언제 발동돼야 하는가? (수동 `/명령`만? 자동 호출도?)
- [ ] **입력/출력 정의** — 무엇을 받아서 무엇을 내놓는가
- [ ] **저장 범위 결정** — 개인용(`~/.claude`) vs 프로젝트용(`.claude`)

### 작성 단계
- [ ] **폴더 생성** — `skills/<skill-name>/`
- [ ] **frontmatter 작성** — `name`, `description`(트리거 기준을 구체적으로!)
- [ ] **본문 지침 작성** — Claude가 따라야 할 단계별 절차, 예시, 주의사항
- [ ] **(선택) 보조 파일 추가** — 스크립트/템플릿/참고문서

### 검증 단계
- [ ] **수동 호출 테스트** — `/<skill-name>` 으로 실행해보기
- [ ] **자동 호출 테스트** — 관련 상황을 만들어 Claude가 스스로 부르는지 확인
- [ ] **지침 다듬기** — 결과를 보고 description/본문 보정
