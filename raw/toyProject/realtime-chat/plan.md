# 실시간 채팅 + 인프라 풀코스 토이프로젝트 — 기획/로드맵 (raw)

> 출처: 토이 프로젝트 기획 대화(2026-06-17). 개발자 기본기 + 관심 인프라 기술을 한 프로젝트에서 단계별로 밟는 게 목표.

## 출발점 — 왜 이 프로젝트
- "개발자라면 꼭 다뤄봤어야 할 내용"으로 토이 프로젝트를 하고 싶음.
- 개발자 핵심 통과의례 5가지: 인증/인가, DB 설계 + 트랜잭션/동시성, 캐싱, 비동기/메시지 큐, 실시간 통신.
- 관심사: AWS 스터디, k8s/Docker 사용, WebSocket. (이 기술들이 뭔지는 아직 잘 모르는 상태에서 시작.)
- 핵심 역량 방향으로 "실시간 채팅" 선택 → WebSocket·동시성·상태관리·인증을 한 번에 밟음.
- 이후 인프라(Docker/AWS/k8s)까지 풀코스로 확장하기로 결정.

## 기술 한 줄 정의 (시작 시 정리)
- Docker: 앱 + 실행환경을 '컨테이너'로 통째 포장. "내 PC에선 됐는데" 문제 해결, 어디서든 똑같이 실행.
- Kubernetes(k8s): 컨테이너 수십~수백 개를 자동 배포·확장·복구하는 관리자. 손으로 관리 불가능해질 때 자동 오케스트레이션.
- AWS: 클라우드 인프라 백화점 (서버 EC2, 저장소 S3, DB RDS, 큐 SQS 등). 서버 안 사고 빌려 씀, 실무 표준.
- WebSocket: 서버↔클라 실시간 양방향 통신 통로. 채팅·알림처럼 "서버가 먼저 말 거는" 기능.
- 학습 사다리: Docker(포장) → docker-compose(여러 개 묶기) → k8s(자동 운영) → AWS(진짜 배포).

## 검토했던 다른 프로젝트 후보
1. 실시간 채팅 + 인프라 풀코스 ← 채택 (관심 기술 4개를 한 프로젝트에서 단계별로)
2. 실시간 서버 모니터링 대시보드 (컨테이너 metrics를 WebSocket 푸시, k8s 개념 체득)
3. 확장 가능한 실시간 알림 시스템 (이벤트 → AWS SQS → 워커 → WebSocket 푸시)
4. 깃 푸시하면 자동 배포되는 미니 PaaS (난이도 높음)

## 스택
- 기본: Node.js + Socket.IO + Vue 3 (이미 익숙한 스택). Spring(STOMP/WebSocket) 대안 가능.

## 핵심 학습 목표 (앱 단)
- WebSocket 양방향 통신: 메시지 송수신, 연결/끊김 이벤트
- 인증/인가: 소켓 핸드셰이크 시 JWT 검증 (HTTP와 다른 포인트)
- 동시성/상태관리: 누가 어느 방에 접속 중인지, 온라인 유저 set 관리
- 영속성: 메시지 DB 저장 + 재접속 시 히스토리 로드
- 확장성: 단일 서버 인메모리 → Redis Pub/Sub로 다중 서버 대응

## 의도적으로 마주칠 함정 (배움 포인트)
- HTTP 인증과 소켓 인증의 차이 (헤더 못 쓰는 경우, 핸드셰이크 타이밍)
- 새로고침 시 연결 끊김/재연결 처리, 중복 메시지 방지
- 단일 서버 인메모리 상태가 다중 서버에서 깨지는 순간 → Redis 도입 동기

## 일정 가정
- 하루 1~2시간(주 ~8~10시간), 앱 개발은 익숙 / 인프라는 처음.
- 총 예상: 약 11~13주 (≈ 3개월). 인프라가 처음이면 AWS/k8s 구간에서 늘어나는 게 정상.

## 주차별 로드맵
- W1~2: 채팅 MVP — WebSocket 양방향, 소켓 JWT 인증 → 1:1 채팅 브라우저 실시간 동작
- W3: 채팅 디테일 + DB — 방(room), 온라인 목록, 입력중 표시, 메시지 영속화 → 새로고침해도 대화 남는 다인 채팅방
- W4~5: Docker — 컨테이너 개념, Dockerfile, docker-compose → 앱+DB+Redis가 `docker compose up` 한 방
- W6~8: AWS 배포 — EC2/ECS, RDS, ElastiCache, 보안그룹, 도메인 → 인터넷에 떠서 친구가 접속 가능 ⭐
- W9: 확장성 준비 — Redis Pub/Sub 어댑터 (서버 2대가 메시지 공유)
- W10~13: Kubernetes — Pod/Service/Deployment, 스케일아웃, 무중단 배포 → k8s 위 자동 운영·복구

## 단계별 완성 지점 (어디서 끊든 결과물 남게)
- W3 끝 → "실시간 채팅 앱을 만들었다" (앱 개발자 결과물)
- W5 끝 → "컨테이너로 포장했다" (Docker 이력)
- W8 끝 → "내가 만든 게 진짜 인터넷에 떠 있다" ⭐ 포트폴리오로 가장 강력
- W13 끝 → "k8s로 운영까지 했다" (인프라 엔지니어 결과물)

## 시간 함정 (미리 알아둘 것)
- AWS(W6~8): 결제카드 등록·IAM 권한·보안그룹 막힘으로 첫 배포에서 하루이틀 헤매는 게 정상. 프리티어 + 비용 알림 필수.
- k8s(W10~13): 개념(Pod/Service/Ingress)이 한 번에 안 들어옴. 로컬(minikube/kind)에서 충분히 굴린 뒤 AWS EKS로.
- 위 일정은 순항 기준 최소치. 막히면 그 주가 1.5배로 늘 수 있음.

## 빠른 길 (시간 쪼들리면)
- W8(AWS 배포)까지를 1차 목표(약 2개월)로 잡고 한 번 완성·발표 → k8s는 시즌2로 분리.
