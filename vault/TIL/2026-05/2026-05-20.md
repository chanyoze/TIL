## Claude Code 스킬 만들기

**배운 것**
Claude Code의 스킬은 `SKILL.md` 하나로 정의되는 "작업 지침서"이며, 폴더에 넣기만 하면 재시작 없이 실시간으로 반영된다.

**핵심 내용**
- 스킬 = `<skill-name>/SKILL.md` (YAML frontmatter + 마크다운 본문)
- frontmatter의 `description`이 **자동 호출의 핵심 기준** → "언제 쓰는지"를 구체적으로 적어야 함
- 본문은 "어떻게 할지" 단계별 지침 + 템플릿 + 작성 원칙
- 저장 위치: 개인용 `~/.claude/skills/`, 프로젝트용 `.claude/skills/`(팀 공유)
- **Live change detection**: 이미 감시 중인 skills 폴더에 스킬을 추가/수정하면 재시작 없이 즉시 반영됨
- `/clear`는 대화 기록만 비우는 것 (재시작 아님)
- 세션 재시작은 `Ctrl+D`로 종료 후 터미널에서 `claude -c`(이어서 재개) 또는 `claude --resume`
- 스킬은 CLI 세션을 스스로 재시작할 수 없다 (Claude가 돌아가는 프로세스 자체이기 때문)

**예시 / 코드**
```markdown
---
name: til-note
description: 오늘 배운 것(TIL)을 날짜별 markdown 노트로 정리할 때 사용
---

# TIL 노트 작성기
(Claude가 따라야 할 단계별 지침...)
```

**더 알아볼 것**
- `allowed-tools`로 스킬이 쓸 수 있는 도구 제한하기
- progressive disclosure: 본문은 짧게, 상세 내용은 별도 파일로 분리해 컨텍스트 절약하기
- 스킬에 보조 스크립트/템플릿 파일 함께 묶기

## 스킬 호출 방식 실습 (수동 vs 자동)

**배운 것**
직접 만든 `til-note` 스킬을 두 가지 방식으로 호출해보며, `description`이 자동 호출을 결정한다는 걸 실제로 확인했다.

**핵심 내용**
- **수동 호출**: `/til-note` 입력 → frontmatter의 `name`이 그대로 슬래시 명령이 됨
- **자동 호출**: "오늘 배운 거 정리해줘"라는 자연어 → Claude가 `description`을 보고 스스로 스킬을 실행함
- 새로 만든 스킬이 **재시작 없이 즉시** 호출됨 → Live change detection 실증
- 같은 날 다시 호출하니 기존 날짜 파일 맨 아래에 새 항목이 **이어 붙음(append)**

**더 알아볼 것**
- `description` 문구를 바꿔가며 어떤 표현일 때 자동 호출이 더 잘 되는지 실험
- 스킬에 인자 전달하기 (예: `/til-note 주제명`)
