빌드·배포 파이프라인(젠킨스 등)이 "성공"으로 끝났는데 막상 접속하면 **흰 화면(blank page)** 만 뜰 때가 있다. 코드 문제가 아니라 **서버 프로세스(컨테이너)가 정상적으로 다시 올라오지 못한** 경우가 많다. 이때 MCP 도구(ims)로 해당 서버를 재시작하면 빠르게 복구된다.

## 증상

- 배포 잡(job)은 초록불(성공)인데, 사이트는 흰 화면 또는 502/응답 없음.
- 새로고침해도 동일. 직전 코드 변경과 무관하게 화면이 안 뜸.

## 원인

- 배포 과정에서 서버 프로세스/컨테이너가 내려간 뒤 **다시 기동되지 않은** 상태.
- 빌드 산출물은 갱신됐지만 이를 서빙하는 런타임이 떠 있지 않아 빈 응답이 나간다.

## 조치 — ims MCP로 재시작

서버에 직접 SSH로 들어가 셸 명령을 치지 않고, **MCP 도구가 제공하는 서비스 카탈로그**를 통해 안전하게 재시작한다.

1. **`list_services`** — 자연어("개발 API 서버")를 정확한 서비스 식별자로 변환. 같은 이름이 dev/prod 양쪽에 있으면 환경을 먼저 구분한다.
2. **`restart_service`** — 해당 서비스를 재시작.
   - **dev**: 즉시 재시작.
   - **prod**: `confirm` 승인이 없으면 거부됨 — 운영은 반드시 명시적 승인 후에만.
3. **`get_restart_status`** — 15~30초 뒤 결과 확인. 컨테이너가 `Restarting → Started` 로 바뀌고 상태가 `Success` 면 복구 완료.

## 핵심

- "배포는 성공인데 흰 화면" → 코드보다 **런타임이 안 떠 있는지**를 먼저 의심한다.
- 재시작은 **카탈로그 기반 도구**로 하면 셸 접근 없이 안전하고, prod는 승인 게이트가 걸려 실수를 막는다.
- MCP(Model Context Protocol): 외부 도구/데이터를 표준화된 방식으로 AI에 연결하는 프로토콜. 여기선 인프라 재시작 명령을 도구로 노출한 것.
