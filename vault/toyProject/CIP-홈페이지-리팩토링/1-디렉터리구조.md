# 2. 디렉터리 구조

```
cip-hp-main/
├─ pom.xml                     # Spring Boot 3.5.14 / Java 21, 의존성 다수(AWS S3, JWT, Tika, MyBatis…)
├─ doc/                        # 기존 마이그레이션 계획 문서 2종 (이번 분석에서 미참조)
├─ src/main/java/com/cloudlab/wt/cip/hp/
│  ├─ CipHpApplication.java
│  ├─ config/SecurityConfig.java          # Spring Security 2-체인 (웹 / REST)
│  └─ modp/                               # 도메인별 패키지 (modp = module-page?)
│     ├─ common/controller/CommonController.java     # "/" index
│     ├─ company/controller/CompanyController.java    # 회사소개 7면
│     ├─ business/controller/BusinessController.java  # 사업분야 3면
│     ├─ solution/controller/SolutionController.java  # 보유솔루션 18면
│     ├─ board/controller/BoardController.java        # 사이버홍보실(게시판) 28면
│     └─ helpdesk/controller/HelpdeskController.java  # 고객센터 6면
└─ src/main/resources/
   ├─ application.yml + application-{local,dev,prod}.properties
   ├─ templates/              # Thymeleaf 뷰 (현 디자인의 핵심)
   │  ├─ index.html
   │  ├─ fragments/           # top, footer, menu01~05, quick, visual  ← 공통 조각
   │  ├─ board/   company/   business/   solution/   helpdesk/
   └─ original-source/        # ★레거시 classic ASP 원본 (참조용 보관)
      ├─ *.asp (게시판 CRUD: news_write/modify/del_action 등)
      ├─ common/ (top_*.asp, bottom.asp, secure.inc.asp …)
      └─ css/ js/ image/ font/ ...
```

> `modp` = 화면(page) 위주 모듈 패키지로 보임. 향후 REST API가 생기면 `moda`(api) 같은 구분이 추가될 여지가 있음.
