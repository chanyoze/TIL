# 3. 계층별 상세 분석

## 3-1. 컨트롤러 — "뷰 리턴 전용"
모든 컨트롤러가 아래 패턴뿐입니다. 파라미터 `Model`은 받지만 **아무것도 담지 않습니다.**

```java
@GetMapping("/board/news")
public String getViewNews(Model aModel) {
    return "board/news";   // 정적 템플릿 그대로 반환
}
```

- `BoardController` 하나에 **28개 메서드** — `news_detail` ~ `news_detail19`(20개),
  `itnews_detail`~`itnews_detail2`(3개), `reference_detail` 등이 **글 1건당 메서드 1개 + 템플릿 1개**로 박혀 있음.
- `helpdesk/recruit_detail`~`recruit_detail3`도 동일 패턴.
- → **게시판이 데이터가 아니라 "페이지"로 존재**. 글이 늘어나면 컨트롤러·템플릿을 계속 추가해야 하는 구조.

## 3-2. 서비스/매퍼/도메인 — 부재
- `service`, `mapper`, `domain`, `dto`, `vo` 패키지 **없음**.
- MyBatis Mapper XML **없음** (`*.xml`은 logback/sitemap/레거시 original-source 뿐).
- `application.yml`엔 Oracle 듀얼 데이터소스(`base`, `ctalk`)가 설정돼 있으나 실제 쓰는 코드가 없음.
- → DB 연동은 `cloudlab-wt-core`가 자동설정을 제공할 가능성이 높지만, **이 프로젝트 자체엔 쿼리 한 줄 없음.**

## 3-3. 보안 (`SecurityConfig`)
- 2개의 `SecurityFilterChain`:
  - `@Order(1)` REST 체인: `/core/api/v*/rest/**`, `/v*/rest/**` → JWT 필터 + 역할(ROLE) 인가.
  - `@Order(2)` 웹 체인: `/**` → 공개 사이트라 대부분 `permitAll()`, `/core/**`·Swagger는 `CLOUDLAB` 롤.
- CSRF disable, CORS 설정, frameOptions disable. 폼로그인/로그아웃 URL은 `EnvVariables`에서 주입.
- 홈페이지(공개)는 인증 불필요 → 디자인 리팩토링엔 영향 적음.

## 3-4. 템플릿 (디자인의 실체) — ⭐ 리팩토링 핵심 대상
**문제점이 집중된 곳입니다.**

1. **테이블 레이아웃 + 고정폭 1180px**: `<table width="1180">` 중첩. CSS Grid/Flex 아님 → 반응형 불가.
2. **인라인 스타일 범벅**: `style="border-bottom:1px solid #ddd; padding-left:15px;"`가 셀마다 반복.
3. **드림위버 시대 잔재**: `MM_showHideLayers()` 자바스크립트로 메뉴 visibility 토글 (각 템플릿에 중복 정의), `onmouseover` 인라인 핸들러.
4. **ASP 스크립틀릿 殘骸(잔해)**: 마이그레이션 누락분이 그대로 남음 — 깨진 마크업.
   ```html
   <input type="hidden" name="page" value="<%=intNowPage%>">   <!-- Thymeleaf가 처리 못 함, 죽은 코드 -->
   ```
   `index.html`, `board/news.html` 등 다수에 존재.
5. **게시판 목록/페이지네이션이 정적**: `news.html`에 20개 행이 손으로 박혀 있고, 페이지네이션은
   `/board/news` → `/board/news1`로 **별도 URL/템플릿**으로 분리됨. 검색창은 주석 처리.
6. **프래그먼트화는 부분적으로 됨** (`th:replace="~{fragments/top :: body}"`):
   `top / footer / menu01~05 / quick / visual`로 공통 조각 분리. **단, 공통 레이아웃(layout decorator)은 없음** —
   각 페이지가 `<head>`·`<body>`를 통째로 들고 fragment를 끌어다 씀(중복 많음).
7. jQuery 1.12 + jQuery-UI, IE 전용 메타(`X-UA-Compatible`), `<meta name="viewport">`는 있으나 레이아웃이 안 따라옴.

## 3-5. 정적 리소스
- `static/css/` (cipsystems.css, common.css, slide.css), `image/`(다수 PNG 버튼·배경), `font/`(나눔폰트 자체호스팅), `down/`(로고 zip).
- 버튼이 전부 **이미지**(`btn_search.png`, `btn_listnext.png`…) → 디자인 현대화 시 CSS 버튼/아이콘폰트/SVG로 대체 대상.
