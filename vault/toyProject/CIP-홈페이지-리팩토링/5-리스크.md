# 6. 리팩토링 시 주의점 / 리스크

- **`original-source/`는 손대지 말 것** — 레거시 동작 명세의 유일한 근거(특히 게시판 CRUD 로직).
- **`cloudlab-wt-core` 의존** — 보안/EnvVariables가 이 모듈에 있어, 빌드엔 GitHub Packages 인증이 필수. 사내 표준이라 함부로 우회 X.
- **죽은 코드 vs 살아있는 코드 구분** — `<%=...%>`, 주석 처리된 검색/쓰기 버튼 등은 제거해도 되지만, fragment 호출/보안 매처는 건드리면 안 됨.
- **고정폭 1180px 가정** — 반응형 도입 시 이미지(배경 `bg_subvisual04.png` 1180px 등)도 함께 손봐야 깨지지 않음.
- **DB 미연동이 정상** — 현재 게시판이 정적인 건 버그가 아니라 "아직 안 만든 것". 동적화는 Phase 3의 별도 작업.
- **커밋/푸시는 사용자 승인 후** (이번 세션 합의).
