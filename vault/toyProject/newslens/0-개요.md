# NewsLens — AI 미디어 큐레이션 플랫폼 *(가제)*

> **가제 · 2026-06-29~** · 백엔드 2인 · 이직 포트폴리오 (테크 · AX)
>
> 외신 AI·테크 뉴스를 자동 수집해 **LLM으로 한국어 번역·요약·태깅**하고, **자연어 RAG 검색**과 **실시간 알림(SSE)** 을 제공하는 미디어 큐레이션 플랫폼. 팀원이 가져온 「AI 미디어 큐레이션」 제안서를 베이스로, 이찬호의 「실시간 채팅 + 인프라 풀코스」 계획을 합쳐 하나의 프로젝트로 정리했다.

## 이 프로젝트는 두 기획의 병합이다

| 출처 | 역할 | 비고 |
|---|---|---|
| 팀원 「AI 미디어 큐레이션 플랫폼」 | **본체 (베이스)** | Spring Boot · pgvector · Redis · Gemini · SSE · 8일 |
| 이찬호 「실시간 채팅 + 인프라 풀코스」 | **흡수 → 일부 시즌2** | 스택을 Spring으로 통일. JWT는 스트레치, 채팅/Redis Pub/Sub/AWS/k8s는 시즌2 |

병합 결과, **8일 코어 = AI 큐레이션(RAG)** 이고, 이찬호의 원래 학습 목표(실시간·인증·인프라)는 **같은 코드베이스 위 개인 시즌2**로 이어진다.

## 문서 구성 (순서대로)

1. **[기존 계획 (원안)](./1-기존계획.md)** — 병합 이전 이찬호의 실시간 채팅 + 인프라 풀코스 계획. "어디서 출발했나" 기록용.
2. **[병합 계획](./2-병합계획.md)** — 현재 기준 계획. 무엇이 겹쳤고/달랐는지, JWT 위치, 업무 분담 재조정안, 도메인·수집·LLM 처리 제안, 8일 일정.
3. **[업무 가이드](./3-업무가이드.md)** — 라인별 상세 태스크·DB 스키마/API/이벤트 계약·체크리스트·브랜치 전략·Day별 진행표.
4. **[팀 협의 안건](./4-협의안건.md)** — 시작 전 팀이 한 번에 합의할 안건(도메인·소스·제안·분담·배포·테스트). 각 안건에 이찬호 제안 첨부.

## 현재 상태 스냅샷

**확정**
- 스택: **Spring Boot** 통일 / 실시간: **SSE** 유지 / **JWT: 스트레치**(설계에 자리만 예약)

**제안 (팀 협의용)**
- 도메인: **AI·테크 동향** / 수집: **The Guardian API**(영문 전문)
- 처리: 번역+요약+태깅을 **1콜로 합침**(전문 번역 X) + 임베딩 1콜 → 기사당 2요청
- RAG: 한국어 요약 임베딩 → 한국어 질문/답변
- 프레임워크: **Spring AI**(무료 티어 연동 검증 완료) · ROME · Flyway · 벡터 **768**차원 · 데모 UI 타임박싱

**업무 분담 (재조정안)**
- 데이터 흐름으로 분리 — **개발자 1: 수집·처리·인프라(쓰기)** / **개발자 2: 조회·검색·실시간(읽기, +인증 스트레치)**. 공통 기반(DB 스키마·Docker·Gemini 래퍼)은 Day1~2 함께. *라인 배정은 팀 협의.*
