# NewsLens, 쉽게 보기

> 전문용어 없이, **비개발자도 이해할 수 있게** 정리한 한 장. 자세한 건 [개요](./0-개요.md)부터.

## 1. 뭐 하는 프로그램이야?

**"외국 IT/AI 뉴스를 자동으로 모아서, AI가 한국어로 번역·요약해주고, 궁금한 걸 물어보면 답해주는 뉴스 비서."**

비유하면 — **똑똑한 신문 스크랩 비서**다.
1. 매일 외국 뉴스를 알아서 긁어오고,
2. 영어를 한국어로 바꿔 3줄 요약을 달아주고,
3. "요즘 AI 업계 무슨 일 있어?" 하고 물으면, 관련 기사들을 찾아 **종합해서 대답**해주고,
4. 새 기사가 들어오면 **새로고침 없이 바로 알림**으로 띄워준다.

## 2. 만들면서 뭘 배울 수 있어?

토이프로젝트지만 "개발자 기본기"를 한 바퀴 도는 게 목적이다.

| 배우는 것 | 쉬운 설명 |
|---|---|
| **AI 붙이기** | 번역·요약을 사람이 아니라 AI에게 시키기 |
| **똑똑한 검색** | 단어가 똑같지 않아도 **뜻이 비슷한** 기사를 찾아주기 |
| **빠른 응답** | 자주 보는 건 미리 저장해 뒀다가 즉시 보여주기 |
| **실시간 알림** | 서버가 먼저 "새 소식 왔어요" 하고 알려주기 |
| **자동화** | 정해진 시간마다 알아서 뉴스 수집 |
| **배포** | 내 컴퓨터가 아니라 인터넷에 올려서 누구나 접속 |

## 3. 기능별로 무슨 역할을 해?

| 기능 | 무슨 일을 하나 | 비유 |
|---|---|---|
| 🗞️ **뉴스 수집기** | 외국 뉴스를 정해진 시간마다 자동으로 가져옴 | 신문 배달부 |
| 🤖 **AI 처리기** | 영어 → 한국어 번역 + 3줄 요약 + 분류(태그) | 번역·요약해주는 비서 |
| 🔎 **질문 검색** | 문장으로 물으면 관련 기사 찾아 종합 답변 | 똑똑한 사서 |
| ⚡ **빠른 보기** | 자주 보는 목록을 미리 담아둬 빠르게 응답 | 자주 쓰는 물건 앞에 두기 |
| 📢 **실시간 알림** | 새 기사 들어오면 바로 화면에 띄움 | 속보 알림 |
| 🔐 **로그인** *(나중에)* | 회원만 쓰게 막기 (지금은 자리만 비워둠) | 출입증 |

## 4. 어떤 기술을 쓰고, 각자 무슨 역할?

이름은 어려워 보여도 **하는 일은 단순**하다. 역할별로 묶어서 본다.

### 🏗️ 앱의 뼈대
| 기술 | 무슨 역할 | 비유 |
|---|---|---|
| **Spring Boot** | 프로그램 본체. 모든 기능이 이 위에서 돌아감 | 건물의 골조 |
| **Gradle** | 필요한 부품(라이브러리)을 모아 조립·실행 | 조립 설명서 + 공구 |

### 💾 데이터 저장
| 기술 | 무슨 역할 | 비유 |
|---|---|---|
| **PostgreSQL** | 수집한 기사(원문·요약·태그)를 영구 보관하는 창고 | 자료 보관 창고 |
| **pgvector** | "뜻이 비슷한 기사"를 찾게 해주는 검색 기능 (창고의 특수 검색대) | 의미로 찾아주는 사서 |
| **Redis** | 자주 보는 목록을 잠깐 담아둬 즉시 응답 | 책상 위 메모지 |
| **Flyway** | 데이터 창고의 구조(설계도)가 바뀐 이력을 관리 | 설계도 변경 기록부 |

### 🤖 AI 두뇌
| 기술 | 무슨 역할 | 비유 |
|---|---|---|
| **Gemini (LLM)** | 번역·요약·질문 답변을 실제로 해주는 AI | 똑똑한 비서의 머리 |
| **text-embedding** | 문장을 숫자로 바꿔 "뜻 비교"가 가능하게 만듦 | 문장의 지문(指紋) |
| **Spring AI** | 위 AI를 앱에 쉽게 연결해주는 도구 | AI 연결 어댑터 |

> 💡 **"똑똑한 검색"의 정체** = pgvector(뜻으로 찾기) + Gemini(찾은 걸 종합해 답하기). 이 조합이 이 프로젝트의 핵심.

### 🖥️ 화면·실시간
| 기술 | 무슨 역할 | 비유 |
|---|---|---|
| **React** | 사용자가 실제로 보는 데모 화면 | 가게의 진열장 |
| **SSE** | 서버가 화면으로 "새 소식!"을 먼저 보내는 통로 | 속보 방송 채널 |
| **Swagger** | 만든 기능들의 사용 설명서를 자동 생성 | 기능 카탈로그 |

### 🚀 운영·배포
| 기술 | 무슨 역할 | 비유 |
|---|---|---|
| **Docker** | 앱+실행환경을 통째로 포장 (어디서나 똑같이 실행) | 이사용 포장 박스 |
| **Railway / Render** | 완성품을 인터넷에 올려 누구나 접속하게 함 | 가게를 낼 상가 자리 |

## 5. 뉴스 한 건이 화면에 뜨기까지 (흐름)

```
① 수집기가 외국 뉴스를 가져온다           (Spring Boot + Guardian)
        ↓
② AI가 한국어로 번역·요약·태그를 단다       (Gemini + Spring AI)
        ↓
③ 창고에 저장한다 (검색용 '지문'도 같이)    (PostgreSQL + pgvector)
        ↓
④ "새 기사 왔어요!" 화면에 알림            (SSE)
        ↓
⑤ 사용자가 보거나 질문하면 빠르게 응답       (Redis + 질문검색)
```

## 6. 어떻게 만들지? (둘이 나눠서)

백엔드 개발자 **2명**이 8일 동안, 일을 둘로 나눠 만든다.

- **한 사람** → *뉴스를 모으고 AI로 가공하는 쪽* (위 흐름 ①②③)
- **다른 사람** → *보여주고 검색·알림 해주는 쪽* (위 흐름 ④⑤)
- 시작 첫 이틀은 **같이** 기본 틀(데이터 저장 형식 등)을 맞춘 뒤 갈라진다.
