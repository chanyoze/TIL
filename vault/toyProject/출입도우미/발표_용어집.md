# 출입도우미 CI/CD 발표 — 용어집 (배포용)

> 발표를 따라오시는 데 도움이 되도록 핵심 용어를 한 줄로 정리했습니다.

## 🔁 CI/CD 기본
| 용어                    | 한 줄 설명                                                        |
| ----------------------- | ----------------------------------------------------------------- |
| **CI** (지속적 통합)    | 코드를 올릴 때마다 **자동으로 빌드·테스트**해 오류를 일찍 잡는 것 |
| **CD** (지속적 배포)    | 통과한 코드를 **자동으로 패키징·배포**하는 것                     |
| **파이프라인**          | 빌드 → 테스트 → 배포를 정해진 순서로 자동 실행하는 흐름           |
| **GitHub Actions**      | GitHub에 내장된 CI/CD 자동화 도구 (워크플로를 YAML로 작성)        |
| **러너 (Runner)**       | 워크플로를 실제로 실행하는 GitHub의 가상 머신                     |
| **워크플로 (Workflow)** | 자동 작업을 정의한 YAML 파일 (예: `ci.yml`, `release.yml`)        |

## 🐘 빌드 / 패키징
| 용어              | 한 줄 설명                                                      |
| ----------------- | --------------------------------------------------------------- |
| **Gradle**        | 빌드·의존성·테스트를 자동화하는 표준 빌드 도구                  |
| **toolchain**     | 쓸 JDK **버전만 선언**하면 경로를 자동으로 찾아주는 Gradle 기능 |
| **Maven Central** | 라이브러리(jar)를 받아오는 중앙 저장소                          |
| **JDK / JRE**     | 자바 **개발 도구 모음** / 자바 **실행 전용** 환경(런타임)       |
| **jpackage**      | jar + JRE를 묶어 **자바 설치 없이** 도는 exe를 만드는 도구      |
| **app-image**     | jpackage 산출물(폴더형 exe+JRE) — 압축 풀고 더블클릭            |

## 🧪 테스트
| 용어                    | 한 줄 설명                                                          |
| ----------------------- | ------------------------------------------------------------------- |
| **JUnit**               | 자바 **단위 테스트** 프레임워크                                     |
| **단위 테스트**         | 함수·모듈 단위로 동작을 자동 검증하는 테스트                        |
| **순수 함수**           | 입력→출력만 있고 화면·파일 같은 부수효과가 없는 함수(테스트가 쉬움) |
| **헤드리스 (Headless)** | 화면·마우스·키보드가 없는 서버 환경 (= CI 러너)                     |

## 🚀 배포 / 릴리스
| 용어                | 한 줄 설명                                                   |
| ------------------- | ------------------------------------------------------------ |
| **태그 (Tag)**      | 특정 커밋에 붙이는 **고정 버전 이름표** (예: `v1.3.0`)       |
| **semver**          | 버전을 `major.minor.patch`로 매기는 규칙                     |
| **GitHub Releases** | 빌드 결과물(zip 등)을 **버전별로** 올려두는 곳               |
| **GitHub Pages**    | 정적 웹페이지를 **무료로 호스팅**하는 기능 (다운로드 페이지) |

## 🧹 리팩토링 / 코드 품질
| 용어            | 한 줄 설명                                               |
| --------------- | -------------------------------------------------------- |
| **리팩토링**    | 동작은 그대로 두고 **코드 구조를 개선**하는 작업         |
| **God Method**  | 한 메서드에 너무 많은 책임이 몰린 안티패턴               |
| **DRY**         | 같은 코드를 반복하지 말라는 원칙 (Don't Repeat Yourself) |
| **관심사 분리** | UI·로직·설정 등을 역할별 모듈로 나누는 설계              |
| **EDT**         | Swing UI를 다루는 **전용 스레드** (UI 작업은 여기서만)   |

## 🌿 Git / 협업
| 용어                     | 한 줄 설명                                              |
| ------------------------ | ------------------------------------------------------- |
| **커밋 / 브랜치 / 머지** | 변경 스냅샷 / 작업 줄기 / 줄기를 합치기                 |
| **PR (Pull Request)**    | 변경을 **검토·병합 요청**하는 단위                      |
| **재현성**               | 누구의 PC·환경에서도 **똑같은 빌드 결과**가 나오는 성질 |
