# TBM B타입 — 설계 고려사항

> 일일안전회의(TBM) B타입 개발 시 정리한 옵션·이관 로직·ERD 고려사항. (내부 제품 식별자 유지, 고객/인명 제거)

## 옵션 정의

| 옵션 | 동작 |
|---|---|
| 옵션70 | 참석 버튼/QR 사용여부 — 둘 다 미사용이면 근로자 참석현황 컬럼 숨김 |
| 옵션73 | 금일 이전 일자 수정가능 여부 |
| 옵션78 | 일일안전회의 타입 — 타입에 따라 공통전달사항 메시지 박스 텍스트 변경 |
| 옵션83 | TBM 구분 사용여부 — 사용 시 교육구분 컬럼 표시 |

## 화면 규칙

- 참석자가 있으면 **점검일자 수정 불가**.
- 그리드 숨김 컬럼은 readonly 처리, 단어사전 반영.
- 일일중점위험·공통전달사항은 A타입과 동일.
- 출력물 서명: **공사팀장 직접 지정** → 지정 사용자 서명만 조회, 미지정 시 서명란 비움.
- 제안자 = 원청 사용자 + 선택한 기업/협력사 사용자. 제안내용 입력 시 제안자 없으면 작성자 자동 지정.

## 근로자의견청취사항 → 위험/제안사항 이관 로직

- **승인 시:** 제안내용 있으면 insert(제안자/위치 유무 무관), 없으면 기존 등록 위험/제안 delete.
  - 매핑: 제안사항→구분, 기업/협력사→기업/협력사, 교육일자→작성일자, 제안자→작성자(없으면 TBM 작성자), 위치→위치, 제안내용→유해위험요인, 공종→공종.
  - 상태: 위험제안사항으로 넘어가면 TBM "제출", 안 넘어가면 "미제출", 넘어간 뒤엔 위험제안사항 상태를 그대로 따름.
- **승인취소 시:** 등록 항목 유지, 재승인 시 update.
- 이관 완료된 TBM은 근로자의견청취사항 **수정 불가**.

## 구조 변경

- TBM 근로자를 **별도 탭으로 분리** + 근로자 첨부파일 추가(ERD에 별도 `FILE_GRP_SN` 추가).
- 원청 승인 시 제안내용 있으면 **제안자·위치·근로자의견별 사진 등록 검증** 추가.
- 근로자참석현황: 팝업 → **탭 분리**.

## 유의 / 백로그

- 위험제안사항 이관 시 **사진도 복사**해서 추가.
- 이관 내용 제안내용 삭제 후 재승인 시 **첨부 파일 테이블(쓰레기 데이터) 동시 삭제** 필수.
- 모바일: TBM 메뉴에서 A/B 타입을 보유 메뉴에 따라 분기해 보여줄지 검토 필요.
