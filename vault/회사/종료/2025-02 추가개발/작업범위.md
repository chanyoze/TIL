# 추가개발 — 작업 범위 정리

> 고객사 추가개발 요청 회의에서 정리한 **신규 메뉴·개선 작업 범위**. (담당자·고객사·타사 식별정보 제거, 기능 범위만 발췌)

## 공통 방향 — 확인(결재) 모듈 분리

- 제출/승인/반려에 대한 **작업자·시간 로그** 요구가 많음 → 기존 결재 프로세스에 영향을 주지 않는 **"확인자" 모듈을 별도 분리**하기로 결정.
- 결재란을 **모듈화**해 원하는 화면에서 원하는 프로세스로 쓰도록 하는 게 목표지만, ① 이미 결재가 박힌 화면 ② 리포트 내 결재란 등 제약이 많아 시간이 걸림.

## 신규 메뉴 (추가개발)

1. **건설장비점검** — 장비 인적사항 기록. 공사일보 연계(공사일보 선택 시 업체 자동), 장비종류·차량번호 **코드 관리**, 제출/승인 프로세스, **QR(출력물에 생성)**, 장비종류별 체크리스트 + 점검 사전 사진 + 가이드 코멘트.
2. **ESH(테마)점검** — 테마점검 + 부적합보고서 성격. 자율 점검(본사/외부/자체), Inspection Category(대/소분류) **사용자 직접 관리**, 작성 주체(본사·현장)와 **조치용 메뉴 분리**, 타 메뉴 연계 없이 간단하게.
3. **3 & 1 OUT** — 본사 규정(원아웃)/현장 규정(쓰리아웃), 근로자 규정 위반 시 **카운트 증가 + 사유 문서화**.
4. **사고보고서** — 6하원칙 텍스트 → **필드 분리**, 필수문서 **카테고리화**(검측요청서 사진대지 방식), **퀵사고보고 ↔ 정식사고보고** 순차 프로세스(4화면), 등록 시 원청 전원 푸시, **사고 대시보드(차트)**, 미등록 근로자 처리.
5. **안전작업허가** — 기존 B타입 출력물과 유사하나 별도 분리.
6. **안전보건점검일지** — ESH + 추가위험발굴 통합(구분코드로 분리).
7. **보건** — 보류.

## 개선 요청

1. **수시위험성평가서 — 특별위험성평가 추가**
   - 회차 중복 등록 처리(월 회차 중복 방지의 부작용 검토), **정규/특별 회차 구분**(정규1~5차 / 특별1~5차), 차수별(0차분·1차분…) 추가 시점 명시.
2. **TBM — 중점 외 내용도 참여**
   - TBM은 작업 기간 내 참석이 맞지만 **안전교육은 교육일자에 구애받지 않도록 옵션 분리**(급한 건).
   - 자사일일중점위험 기본 데이터: 회의록 + 수시 + 일일안전회의등록으로 확대(중복 키는 회의록 우선).
