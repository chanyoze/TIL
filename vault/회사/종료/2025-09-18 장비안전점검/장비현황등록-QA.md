# 장비현황등록 — 1차 QA 정리

> 장비현황등록(`CeqpRegProjectEquipmentStatus`) 화면 1차 QA 피드백. (내부 화면ID 유지, 인명/고객 없음)

## 데이터 / 표기

- 조직 `DeptNo` 관련 항목 삭제.
- **장비사양 그리드 헤더에도 마우스 오버 시 툴팁** 조회되도록.
- **최근사용이력** 정리:
  - 사용여부는 최근사용이력과 무관 → **별도 분리**.
  - "기업/협력사" 컬럼이 *사용기간의* 기업/협력사인지 *운전자 소속*인지 혼동 → 컬럼명 정리.
  - 결정: `최근사용이력 → 사용이력`, `사용기간 내 기업/협력사 → 사용회사`, `운전자이력 내 기업/협력사 → 삭제`(한 현장 내 통합 관리라 소속 구분 불필요).
  - `Sysdate` vs 사용일자 검증 없음.

## 화면 / UX

- 조회조건에 **장비종류별 필터** 추가 검토.
- 그리드 헤더 `style: height:20px` 삭제(헤더가 길어짐).
- 파일 분리: `CeqpRegProject…` → 원청(`CeqpRegCompany…`)·협력사(`CeqpRegCooperation…`)로 **분리**.
- 정렬: 최근 생성 건이 위로 오도록 **순번 내림차순**.

## 일괄 처리

- **여러 건 동시 삭제** 지원 — 그리드 옵션 `DataDragSelect / UseCtrlKey / useShiftKey / bMultiDel` 적용.
- **여러 건 동시 사용여부 변경** 검토 — 체크박스 다중 선택 후 커스텀 버튼으로 `USE_TF`만 업데이트하는 트랜잭션 제공 방안.
