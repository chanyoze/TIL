# HSE점검 — 설계 검토 메모

> HSE점검 모듈 설계 시 검토한 쟁점. (내부 제품 맥락 유지, 인명/고객 없음)

## 검토 쟁점

1. **점검자 복수 등록 필요?**
   → 필요. **별도 테이블** 생성, 노출 방식은 문서관리 공유협력사 방식 UI를 차용(UI 변경 필요).
2. **부적합 등급 세부 분류?** → 확인 필요.
3. **현장담당자의 역할** (기록용인지 기능 수행인지)
   → 푸시 받는 역할인데 결재 요청자와 역할이 너무 중복 → 우선 삭제, **현장 직책으로 푸시 발송**.
4. **건설중장비 안전점검 시 장비현황등록 장비와 연결?**
   → 확인 필요. (검측요청서처럼 추가 시 상세 팝업 분리하는 방식 검토)

## 메모

- 코드등록 탭으로 분리(부모-자식 관계로 보임).
- 화면설계 목표: ~10/2.
