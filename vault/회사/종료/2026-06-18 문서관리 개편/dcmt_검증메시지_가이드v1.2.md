# 문서관리 검증 메시지 가이드 (팀 공유용)

> 문서관리 6개 화면에서 **사용자 동작을 차단할 때 노출되는 메시지·조건**을 정리한 기준 문서입니다.
> 인터뷰(2026-06-09)로 문구/조건을 통일한 최종본입니다. 신규/수정 시 이 표를 기준으로 맞춰주세요.

## 대상 화면
| 약칭 | 파일 | 설명 |
|---|---|---|
| 원청 | `DcmtRegPrjtDcmt` | 원청 문서등록 |
| 원청T2 | `DcmtRegPrjtDcmtType2` | 원청 문서등록(Type2) |
| 협력 | `DcmtRegPctrtDcmt` | 협력사 문서등록 |

| 원청버전 | `DcmtPopRegPrjtDcmt` | 원청 문서 버전 팝업 |

## 상태코드 범례
`10` 등록중/등록완료 · `20` 승인대기 · `30` 승인중 · `40` 승인반려 · `50` 승인완료

> 공통 가드 `!~msg.check.beforeSaveNotiMod~!` = **저장하지 않은 변경분이 있을 때** 진행 차단(프레임워크 표준). 각 저장/버튼에서 `cips.isAllDataModify()`로 공통 사용.

## 설계 메모 (의도된 차이 — 변경 대상 아님)

1. **삭제권한 출처가 화면별로 다름** — `DOC_DEL_PSBL_TF` 컬럼명은 같지만 의미가 다름:
   - **원청(PRJT)**: 사용자 **보안등급** 기준 (`wdlCdDelAuthVer`)
   - **협력(PCTRT)**: **현장 옵션 SFAS_OPT_119** 기준 (`wdlCdOpt`)
   - → 메시지도 "보안등급" vs "현장"으로 의도적으로 다름.
2. **협력사 화면엔 연결문서 개념이 없음** → 삭제 메시지 대상명을 "문서"로 고정(원청은 {문서/연결문서} 동적).
3. **"본사 관리자" 문구** — 현행 유지(원청/협력 공통 안내).

## 검증 순서 (우선순위)

> if-else-if 체인은 **위에서 먼저 걸리면 아래 메시지는 안 나옴**. 순서는 "막는 사유의 근본성" 기준으로 **권한/소유 → 문서속성 → 워크플로** 순으로 통일.
> ⚠️ 순서는 "어떤 메시지가 뜨는지"만 결정 — **차단 여부(가능/불가) 결과는 순서와 무관**.

**삭제 표준 순서**
1. 삭제권한 (보안등급/현장 `DOC_DEL_PSBL_TF`)
2. 원청공유 (`REG_COMPANY_ID != 내 회사`) — 협력만
3. 수정권한 (`UPD_PSBL_TF != 'T'`) — 협력만
4. 시스템카테고리 (`SYS_CAT_TF`)
5. 상태 (`[20,30,50]`)
6. 버전 (`VER_DEL_PSBL_TF`)

> 원청/원청T2엔 2·3(원청공유·수정권한)이 없어 `1→4→5→6`. 협력버전팝업은 `원청공유 → 수정권한`만.

**버전갱신 표준 순서**
1. 수정권한 (`UPD_PSBL_TF`) — 협력만
2. 시스템카테고리
3. 상태

> 원청/원청T2엔 수정권한 없어 `시스템 → 상태`.

## 변경 이력
