# 문서관리 6개 화면 — 검증(동작 차단) 메시지 정리

> 목적: 6개 화면에서 **메시지로 동작을 막는 검증**을 한곳에 모아, 조건·문구를 비교하고 일관성/정합성을 인터뷰로 확정하기 위함.
> ⚠️ 표시는 **유사 동작인데 문구/조건/처리가 상이**하여 검토가 필요한 항목.

대상 화면
1. `DcmtRegPrjtDcmt` — 원청 문서등록
2. `DcmtRegPrjtDcmtType2` — 원청 문서등록(Type2)
3. `DcmtRegPctrtDcmt` — 협력사 문서등록
4. `DcmtPopRegDcmtConfirm` — 문서 승인 팝업
5. `DcmtPopRegPrjtDcmt` — 원청 문서 버전 팝업
6. `DcmtPopRegPctrtDcmt` — 협력사 문서 버전 팝업

> 공통 가드: `!~msg.check.beforeSaveNotiMod~!` = **저장하지 않은 변경분이 있을 때** 진행 차단(프레임워크 i18n). 여러 버튼/저장에서 `if(cips.isAllDataModify())`로 공통 사용 → 개별 표기 생략.

## ⚠️ 검토 필요 — 유사 동작인데 상이한 항목

### A. 삭제권한 없음 — 메시지/출처 다름
- **PrjtDcmt/Type2**: "삭제 권한이 없는 **보안등급**입니다…" (출처 `wdlCdDelAuthVer.DOC_DEL_PSBL_TF`, 사용자 보안등급)
- **Pctrt**: "삭제 권한이 없는 **현장**입니다…" (출처 `wdlCdOpt.DOC_DEL_PSBL_TF`, 현장 옵션 SFAS_OPT_119)
- 권한 모델 자체가 다름(보안등급 vs 현장옵션). 의도된 차이인지 확인 필요. + 셋 다 "**본사** 관리자" 문구 → "원청"으로 통일 여부?

### B. 버전갱신 불가(시스템카테고리) — 조사 차이
- "버전갱신**을 할** 수 없습니다" (PrjtDcmt 1307 / Pctrt 908) vs "버전갱신 할 수 없습니다" (Type2 1395)

### C. 버전갱신/문서이관/카테고리변경 불가(상태) — 어미 차이
- "{상태명} **상태의 문서는** …" (PrjtDcmt / Pctrt) vs "{상태명}**인 상태는** …" (Type2)
- 또한 버전갱신 상태조건: PrjtDcmt/Pctrt는 **차단 `[20,30,40]`**, Type2는 **허용 `[10,50]`** 로 표현(실효 동일하나 표현 불일치)

### D. 연결문서 추가 불가(시스템카테고리) — 조사 차이
- "문서**에는** 연결문서를…" (PrjtDcmt 622) vs "문서**는** 연결문서를…" (Type2 645)

### E. 접근권한 체크 — 훅 위치 다름 (동작 검토 필요)
- **PrjtDcmt/Type2**: `retrieve.onClickAfter` + `PC_CD == "C"` → return false
- **Pctrt**: `retrieve.onValidationAfter` + `PC_CD != "C"` → return false
- `onClickAfter`는 **조회가 끝난 뒤** 호출이라 return false로 실제 차단이 되는지 의문(조회는 이미 수행). 의도/동작 확인 필요.

### F. 삭제 메시지의 대상 명칭(`sDocTypeName`)
- PrjtDcmt/Type2: "{문서/연결문서}는 삭제…" (동적, 연결문서 그리드 존재)
- Pctrt: "문서는 삭제…" (고정) — 협력사 화면엔 연결문서 그리드 없음(의도일 수 있음)

### G. 시스템카테고리 삭제 메시지 — 대상 명칭
- PrjtDcmt/Type2: "{문서/연결문서}는" (sDocTypeName) vs Pctrt: "문서는" (고정)

### H. 버튼 가드 `return` 일관성 (Confirm 팝업)
- 승인취소(wbtnCfmCanc)만 `return false`, 나머지(승인요청/요청취소/승인/반려)는 `return` — onclick 핸들러라 기능은 동일하나 일관성 부족.

### I. "원청 현장에서 공유한 문서는 삭제할 수 없습니다" — 화면별 중복
- Pctrt(522) / 버전팝업 PopRegPctrt(363) 동일 문구. (정상, 참고용)
