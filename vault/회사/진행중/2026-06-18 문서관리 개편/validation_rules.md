# 검증 규칙 문서

## 버전 갱신
1. **상태코드 검증**
   - 상태코드가 `'10'`, `'40'`, `'50'` 일 경우에만 버전 갱신 가능.
   - 버전 갱신 시, 상태코드를 `10` 으로 초기화하고 승인요청순번도 초기화.

## 문서 이관
1. **상태코드 검증**
   - 상태코드가 `'10'`(등록완료) 혹은 `'40'`(승인반려)인 경우에만 문서 이관 가능.
   - 문서 이관 시, 상태코드를 `10` 으로 초기화하고 승인요청순번도 초기화.

## 문서 등록
1. 원청, 협력, 승인 사용하는 카테고리로 문서 등록 시에만 승인요청순번을 채번.
2. 원청 → 연결문서, 참여협력사, 공유현장 지정 가능

## 문서 수정
1. **1차 검증**: 카테고리 변경 여부 확인 (`prevCatSn != nowCatSn`).
2. **2차 검증**: 승인라인 저장 여부 확인 (TDCMT_DCMT 테이블에 `CFM_DEM_SN` 매핑된 경우).
3. **3차 검증**: 승인요청을 하지 않았을 경우 – 한 명이라도 승인 진행했을 시 카테고리 변경 자체를 차단.
   - `deleteSanctnStepLine` 실행
   - `deleteSacntnStep` 실행
   - `updateCfmInfoReset` 문서 승인 매핑 초기화 실행

## 문서 삭제
1. 최종 버전 문서 상태 검증 – 승인대기(20), 승인중(30), 승인완료(50)일 경우 삭제 불가, 메시지 반환.
2. 버전 문서 상태 검증 – 버전 중에 하나라도 승인완료가 있으면 삭제 불가, 메시지 반환.
3. 현장문서등록 > 문서설정 옵션 `'사용안함'` 시 삭제 불가, 메시지 반환.
4. 사용자 보안등급 – `DOC_DEL_PSBL_TF`가 `F` 인 경우 무조건 삭제 불가, 메시지 반환.

**위 검증 모두 통과 시 삭제 진행**
- 반복문 사용하여 단계별 삭제 수행:
  1. 버전들의 승인라인 단계 일괄 삭제
  2. 버전의 승인라인 일괄 삭제
  3. 최종 버전 문서의 승인라인 단계 삭제
  4. 최종 버전 문서의 승인라인 삭제
  5. 기존 `delete` 문 실행

## 참여협력사
1. 작성회사가 협력사일 경우 해당 협력사 참여협력사에서 선택 해제 불가.
2. patch: 접근 가능한 회사인지 확인 검증.

## 공유현장
1. patch: 접근 가능한 현장인지 확인 검증.
2. select: 진행 중인 현장, 사용 중인 현장 여부 확인.
