# 문서관리 개편 — 배경 & 설계 리뷰

> 도입 고객사가 기존의 이원화된 문서 결재 체계를 쓰다가 본 솔루션으로 전환하면서 요청한 개선사항을 바탕으로 한 개편 설계 리뷰.
> (고객/현장 식별정보는 익명화) — 같은 폴더의 [검증 메시지 가이드](dcmt_검증메시지_가이드v1.2) · [검증 규칙](validation_rules)이 이 설계의 "결과물"이다.

## 요청 배경 (고객 요구사항)

1. **문서 작성주기 관리** — 카테고리별로 작성주기를 두고 지켜지는지 보고 싶다.
2. **문서 결재(승인) 기능** — 문서마다 결재가 필요하다.
3. **작성주기별 모니터링** — 현장별로 주기에 맞게 작성 중인지 한눈에.
4. **삭제 권한 통제** — 퇴사자/협력사 직원이 공유 문서를 지우고 나가 곤욕을 치른 적 있음 → 작성 권한자도 삭제 못 하게.
5. **본사→전 현장 문서 공유** 방법.

---

## 1. 문서 작성주기 관리

- 카테고리에 **작성주기**를 매핑. 주기 종류는 **시스템 공통 코드**로 제공: `일간·주간·월간·분기·반기·연간` 6종.
- "매주 둘째 금요일" 같은 **가변 캘린더**도 검토했으나, 모니터링 집계 난이도·시스템 복잡도가 급증하고 사용자도 활용이 어려워 **고정 주기로만** 제공.
- 주기에 맞게 미등록 시 **다음 날 오전 9시 스케줄링 푸시 1회**, 대상은 현장 사용자 전원. (잦은 푸시로 인한 피로도 고려)
- **열린 논점:** 더 강한 강제성이 필요한가? (오늘 작성해야 할 문서를 모아 보는 TODO/대시보드 위젯 등)

## 2. 문서 승인(결재) 프로세스

- 요청: **역전결**(순서 무관 아무나 승인) + **반려 시 일괄 반려**.
- 이전 프로젝트에서 구현한 승인 프로세스를 **시범 차용** — 문서관리를 타 메뉴 확산의 첫 케이스로.
- 단, 승인라인 등록 메뉴가 안전보건 시스템에 묶여 있어 **타 메뉴에서도 쓸 수 있게 분리** 필요.
- **기존 UX 보호:** 전 문서에 갑자기 승인을 강제하면 혼란 → **카테고리 단위로 "승인 사용" 지정**. 승인 필요한 문서만 승인사용 카테고리로 등록.

## 3. 작성주기별 모니터링

- 화면 2종 신설: **현장용**(현장 단위), **본사용**(전 현장).
- **동적 렌더링** 기조 — 주기 종류가 평생 6개로 고정된다는 보장이 없으므로 화면을 일일이 만들지 않고 동적으로. 조회조건도 동적.
- **현장용:** 그리드 1개 **단일 매트릭스 뷰**(카테고리 × 일자).
- **본사용:** 좌측 카테고리 / 우측 사용 현장·작성여부 **마스터-디테일 스플릿 뷰**.
- "완료(O)" 기준은 **승인 여부와 무관하게 작성 여부**로 판단.
- 렌더 속도 위해 조회조건에 **리미트**.

## 4. 문서 삭제 권한 관리

- 요청: 작성 권한자도 삭제 불가, **본사 관리자만** 삭제.
- 전체 메뉴에 일괄 적용은 방침/현실상 무리 → **기존 보안등급에 삭제 권한 제어 로직을 결합**.
- 기본값은 기존과 동일(영향 없음), 통제를 원하면 **삭제가능여부 체크 해제**로 제어. 보안등급은 사용자등록에서 이미 매핑되어 있어 수정 최소화.

## 5. 본사 → 현장 문서 공유

- **안전보건관리체계** 구현 때 만든 본사 문서등록 선례를 차용.
- 본사 관리자만 접근, 작성된 문서는 기존 문서등록 화면에서 조회.
- (검토 중) 협력사에도 조회가 필요한지.

---
