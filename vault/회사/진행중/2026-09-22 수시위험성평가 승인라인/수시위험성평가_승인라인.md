# 수시위험성평가 승인라인 시퀀스 다이어그램

## 승인 단계 구성

| 순서 | 단계 | 승인자구분 |
|------|------|----------|
| 1번째 | 요청 | 작성자 |
| 2번째 | 공사팀장승인 | 공사팀장 |
| 3번째 | 안전팀장승인 | 안전팀장 |
| 4번째 | 대표승인 | 대표 |

---

## Case 1. 협력사가 있는 경우 (원하도급, 일반적)

> 참여자: 본사(회사) · 현장 · 협력사

```mermaid
sequenceDiagram
    participant 본사 as 본사 (회사)
    participant 현장
    participant 협력사

    본사->>현장: ① 승인라인 등록
    note over 본사,현장: 공사팀장→안전팀장→대표 4단계 라인 사전 설정

    협력사->>협력사: ② 수시위험성평가 작성
    협력사->>현장: ② 요청 단계 승인 (1번째)
    note over 협력사,현장: 작성자(협력사) 본인 승인 처리

    현장->>현장: ③ 공사팀장 승인 (2번째)
    현장->>현장: ④ 안전팀장 승인 (3번째)
    현장->>현장: ⑤ 대표 승인 (4번째)
    현장-->>협력사: 승인 완료 알림 발송
    현장-->>본사: 승인 완료 알림 발송
```

---

## Case 2. 협력사가 없는 경우 (원도급, 예외사항 - 협력사가 없을 때)

> 참여자: 본사(회사) · 현장

```mermaid
sequenceDiagram
    participant 본사 as 본사 (회사)
    participant 현장

    본사->>현장: ① 승인라인 등록
    note over 본사,현장: 공사팀장→안전팀장→대표 4단계 라인 사전 설정

    현장->>현장: ② 수시위험성평가 작성
    현장->>현장: ② 요청 단계 승인 (1번째)
    note over 현장: 현장 작성자 본인 승인 처리

    현장->>현장: ③ 공사팀장 승인 (2번째)
    현장->>현장: ④ 안전팀장 승인 (3번째)
    현장->>현장: ⑤ 대표 승인 (4번째)
    note over 현장: 최종 승인 완료
```

# 수시위험성평가 승인라인 플로우 차트


> **반려** → 현재 회차를 보존하고, 다음 회차를 새로 생성하여 요청자부터 재시작  
> **승인취소** → 자신의 승인을 철회하고 바로 이전 단계로 복귀

```mermaid
flowchart TD
    S(협력사/현장) --> CREATE

    CREATE[위험성평가서 생성] --> REQ

    REQ{요청자 결재}
    REQ -- 승인 --> CM

    CM{공사팀장 결재}
    CM -- 승인 --> SM
    CM -- 승인취소 --> REQ
    CM -- 반려 --> REQ

    SM{현장소장 결재}
    SM -- 승인 --> CEO
    SM -- 승인취소 --> CM
    SM -- 반려 --> REQ

    CEO{대표 결재}
    CEO -- 승인 --> DONE
    CEO -- 승인취소 --> SM
    CEO -- 반려 --> REQ

    DONE[최종 승인 완료]
    DONE --> END(푸시 알림 발송)

    style S fill:#000, stroke:#000
    style END fill:#fff, stroke:#000, stroke-width:4px
```
